# Branch convention
  - **develop**: 개발 브랜치
    - **feature**: 기능 개발 브랜치
    - **refactor**: 리팩토링 브랜치
  - **stage**: 스테이징 브랜치
  - **deploy**: 배포 브랜치
  - **ex)**
    - [Branch Name]/[Issue Number] 형식으로 생성
    - HotFix의 경우는 develop 브랜치 에 바로 반영
    - ex) feature/001, refactor/002
---
# Commit convention
  - feature/[word] : 새로운 기능 추가.
  - refactor/[word]: 버그가 아닌 코드의 리팩토링 및 수정.
  - fix/[word]: 의도하지 않았던 동작(버그) 수정.
  - docs/[word]: 문서 업데이트.
  - env/[word]: 환경 설정 업데이트.
  - **ex)**
    - feature/ 로그인 기능 개발
    - docs/ README Update
    - fix/ DTO 검증 에러 수정
